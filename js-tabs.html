Подробнее

Отличительной особенностью реализации табов является возможность размещения тегов кнопок/контента одной группы в разных частях web-страницы, как показала практика, web-проекты с такой особенностью тоже существуют.

Компактность

Размер основных/минимальных файлов HTML/CSS/JS скрипта (ядра табов) в минифицированном виде с помощью сборщика проектов webpack - HTML((122 байта - минимум когда 2 вкладки) + (52 байта за добавочную 1 вкладку) ) + CSS(153 байта) + JS(728 байта) = 1003 байта = почти ровно 1Кб. Это крайне компактно. Для сравнения, только подключение вспомогательного инструмента JQuery для запуска множества плагинов, которые без него не работают jquery-2.1.4.min.js = 83Кб.
Компактность важный параметр, чем меньше размер файлов скрипта (табов) тем:

  1. быстрее загрузка страницы
  2. лучше показатели SEO у поисковиков
  3. выше степень юзабилити
  4. меньше расход трафика

Кроссбраузерность

Ядро табов (основа, без визуального оформления, только функциональность) отмечена комментариями в коде CSS, JS, разработана с учётом кроссбраузерности, поддержкой даже старых браузеров таких как IE7+.

Ядро табов:

  1. пользовательский атрибут у кнопки data-tabs="group-1 btn-1" и соответствующий ей контент с атрибутом data-tabs="group-1 panel-1" значение которого состоит из двух слов, первое - группа табов (group-1, название настраивается программистом) второе - определяет кнопку btn- или контент panel-, после дефиса вместо единицы может быть одинаковое название (для пары кнопка\контент определенной группы) настраеваемое программистом
  2. класс tabs__act добавляется в ручную активной кнопки и соответствующему этой кнопки контенту
